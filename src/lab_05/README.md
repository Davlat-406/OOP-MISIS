## О проекте

Данная лабораторная работа является продолжением ЛР-1, ЛР-2, ЛР-3. В проект добавлена поддержка **функций высшего порядка**, **стратегий сортировки и фильтрации**, **callable-объектов** и **цепочек операций**. Коллекция `ServerCollection` расширена методами `sort_by()`, `filter_by()` и `apply()`, которые принимают функции как аргументы, что делает код гибким и переиспользуемым.

---

## Цель работы

- Освоить передачу функций как аргументов в другие функции и методы
- Научиться применять встроенные функции высшего порядка: `map`, `filter`, `sorted`
- Понять концепцию паттерна «Стратегия» и реализовать его на Python
- Освоить lambda-выражения и их практическое применение
- Интегрировать функциональный стиль с объектно-ориентированным кодом из предыдущих ЛР

## Описание стратегий и функций

### Стратегии сортировки (strategies.py)

| Функция | Что возвращает | Порядок сортировки |
|---------|----------------|-------------------|
| `by_name(item)` | `item.name` | Алфавитный (А→Я) |
| `by_ip(item)` | `item.ip` | Алфавитный по IP |
| `by_connections(item)` | `item.connections` | Числовой (от меньшего к большему) |
| `by_status(item)` | 0, 1, 2, 3 | Online → Offline → Maintenance → неизвестный |
| `by_name_then_connections(item)` | `(item.name, item.connections)` | Сначала по имени, потом по подключениям |

---

### Функции-фильтры (strategies.py)

| Функция | Условие | Что отбирает |
|---------|---------|-------------|
| `is_online(item)` | `item.status == "Online"` | Только онлайн серверы |
| `is_active(item)` | `item.active == True` | Только активные серверы |
| `is_game_server(item)` | `isinstance(item, GameServer)` | Только игровые серверы |
| `is_database_server(item)` | `isinstance(item, DatabaseServer)` | Только серверы БД |

---

### Фабрики функций (strategies.py)

| Фабрика | Что делает | Пример использования |
|---------|-----------|---------------------|
| `make_status_filter(allowed_statuses)` | Создаёт фильтр для заданных статусов | `filter = make_status_filter(["Online", "Maintenance"])` |
| `make_connections_filter(min_conn, max_conn)` | Создаёт фильтр для диапазона подключений | `filter = make_connections_filter(10, 50)` |

---

### Callable-объекты (strategies.py)

| Класс | Метод `__call__` | Что делает |
|-------|-----------------|------------|
| `DiscountStrategy(percent)` | `__call__(item)` | Возвращает строку о применении скидки |
| `UpgradeStrategy(upgrade_type)` | `__call__(item)` | Улучшает сервер (меняет `_max_connections`) |
| `ReportStrategy()` | `__call__(item)` | Возвращает словарь с данными сервера |

---

### Методы коллекции (collection.py)

| Метод | Параметры | Что делает | Возвращает |
|-------|----------|-----------|-----------|
| `sort_by(key_func, reverse=False)` | Функция-ключ, порядок | Сортирует коллекцию | `self` (для цепочек) |
| `filter_by(predicate)` | Функция-условие | Фильтрует коллекцию | Новая коллекция |
| `apply(func)` | Функция или callable-объект | Применяет функцию к каждому элементу | Список результатов |

## Демонстрация работы

В файле `demo.py` реализованы следующие сценарии:

1. **Создание коллекции** — 9 объектов (обычные серверы, игровые серверы, серверы БД)

7. **Callable-объекты** — `DiscountStrategy`, `UpgradeStrategy`
8. **Цепочка операций** — `filter_by().sort_by().apply()`
9. **Замена стратегии** — один метод, разные результаты
