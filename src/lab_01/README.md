## О проекте

Данная лабораторная работа представляет собой реализацию класса **Server** на языке Python с применением принципов объектно-ориентированного программирования: инкапсуляция, свойства, магические методы, а также продвинутые концепции (отдельная валидация, логические состояния, изменение поведения в зависимости от состояния).

---

## Цель работы

- Освоить объявление пользовательских классов
- Разобраться с инкапсуляцией (закрытые поля, контроль доступа)
- Научиться использовать свойства (@property)
- Переопределить магические методы (__str__, __repr__, __eq__)
- Понять разницу между атрибутами класса и экземпляра
- Реализовать валидацию отдельными методами
- Создать объекты с изменяемым состоянием и поведением

## Класс Server

### Атрибуты экземпляра (закрытые поля)

| Атрибут | Тип | Описание |
|---------|-----|----------|
| name | str | Имя сервера |
| ip | str | IP-адрес |
| status | str | Статус (online/offline/maintenance) |
| connections | int | Текущее количество подключений |
| active | bool | Активен ли сервер |

### Атрибуты класса (общие для всех экземпляров)

| Атрибут | Описание |
|---------|----------|
| total_servers | Счётчик созданных серверов |
| max_connections | Максимальное число подключений (1000) |

## Логические состояния

### Состояние 1: Активен / Неактивен

| Состояние | Описание | Ограничения |
|-----------|----------|-------------|
| Активен | Сервер активен | Можно подключаться, пинговать |
| Неактивен | Сервер неактивен | Нельзя подключаться, нельзя пинговать |

### Состояние 2: Статус сервера

| Статус | Описание | Разрешены подключения? |
|--------|----------|------------------------|
| online | В сети | Да |
| offline | Не в сети | Нет |
| maintenance | Обслуживание | Нет |

### Состояние 3: Количество подключений

| Диапазон | Поведение |
|----------|-----------|
| 0 <= connections < max_connections | Можно подключаться |
| connections = max_connections | Достигнут лимит, подключение запрещено |

---

## Валидация данных

| Поле | Что проверяется |
|------|-----------------|
| name | Тип str, не пустое |
| ip | Формат xxx.xxx.xxx.xxx, числа от 0 до 255 |
| status | Только online/offline/maintenance |
| connections | Число >= 0, не превышает max_connections |

## Демонстрация работы

![](/images/lab_01/demo_01.png)
![](/images/lab_01/demo_02.png)
В файле `demo.py` реализованы следующие сценарии:

1. **Создание серверов** — демонстрация конструктора
2. **Вывод через print** — проверка __str__
3. **Вывод через repr** — проверка __repr__
4. **Атрибуты класса** — доступ через класс и экземпляр
5. **Изменение статуса** — проверка сеттера
6. **Подключения** — бизнес-методы connect() и disconnect()
7. **Состояния** — активация/деактивация сервера
8. **Ограничения** — проверка запрета деактивации с подключениями
9. **Сравнение** — проверка __eq__
10. **Валидация** — обработка некорректных данных через try/except
11. **Перезагрузка** — метод restart()
12. **Полный цикл** — жизненный цикл сервера
