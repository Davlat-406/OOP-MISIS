## О проекте

Данная лабораторная работа представляет собой реализацию класса **Server** на языке Python с применением принципов объектно-ориентированного программирования: инкапсуляция, свойства, магические методы, а также продвинутые концепции (отдельная валидация, логические состояния, изменение поведения в зависимости от состояния).

Валидация данных вынесена в отдельный модуль `validate.py`, что обеспечивает чистоту кода и возможность повторного использования.

## Класс Server

### Атрибуты экземпляра (закрытые поля)

| Атрибут | Тип | Описание |
|---------|-----|----------|
| _name | str | Имя сервера |
| _ip | str | IP-адрес |
| _active | bool | Активен ли сервер |
| _connections | int | Текущее количество подключений |
| _status | str | Статус (Online/Offline/Maintenance) |
| _max_connections | int | Максимальное число подключений |

---

## Функции валидации

Модуль `validate.py` содержит отдельные функции для проверки корректности данных:

| Функция | Что проверяет |
|---------|---------------|
| validate_name() | Тип str, не пустое, длина от 3 до 50 символов |
| validate_ip() | Формат xxx.xxx.xxx.xxx, числа от 0 до 255 |
| validate_connections() | Число >= 0, не превышает лимит 1000 |
| validate_status() | Только Online/Offline/Maintenance |
| validate_active() | Тип bool |

## Логические состояния

### Состояние 1: Активен / Неактивен (active)

| Состояние | Описание | Ограничения |
|-----------|----------|-------------|
| active = True | Сервер активен | Можно подключаться, пинговать |
| active = False | Сервер неактивен | Нельзя подключаться, нельзя пинговать |

### Состояние 2: Статус сервера (status)

| Статус | Описание | Разрешены подключения? |
|--------|----------|------------------------|
| Online | В сети | Да |
| Offline | Не в сети | Нет |
| Maintenance | Обслуживание | Нет |

### Состояние 3: Количество подключений (connections)

| Диапазон | Поведение |
|----------|-----------|
| 0 <= connections < max_connections | Можно подключаться |
| connections = max_connections | Достигнут лимит, подключение запрещено |

## Валидация данных

Все данные проходят проверку через функции из модуля `validate.py`:

- **name**: строка, не пустая, длина 3-50 символов
- **ip**: строка в формате IPv4 (четыре числа от 0 до 255, разделённые точками)
- **active**: булево значение (True/False)
- **connections**: неотрицательное целое число, не превышающее max_connections
- **status**: строка, одно из значений: "Online", "Offline", "Maintenance"
- **max_connections**: положительное целое число (в конструкторе)

## Демонстрация работы

В файле `demo.py` реализованы следующие сценарии:

1. **Создание серверов** — демонстрация конструктора
2. **Вывод через print** — проверка __str__
3. **Вывод через repr** — проверка __repr__
4. **Атрибуты экземпляра** — доступ к max_connections
5. **Изменение свойств** — проверка setter'ов с валидацией
6. **Подключения** — бизнес-методы connect() и disconnect()
7. **Состояния** — проверка ping() на активном и неактивном сервере
8. **Перезагрузка** — метод restart()
9. **Сравнение** — проверка __eq__
10. **Валидация** — обработка некорректных данных через try/except
11. **Полный цикл** — жизненный цикл сервера
