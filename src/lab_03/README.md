## О проекте

Данная лабораторная работа является продолжением ЛР-1 и ЛР-2. На основе существующего класса **Server** построена иерархия классов с использованием механизма наследования. Созданы дочерние классы, которые добавляют новые атрибуты и методы, а также переопределяют поведение родительских методов.

---

## Цель работы

- Освоить механизм наследования классов
- Научиться строить иерархию объектов
- Понять разницу между базовым и производным классами
- Научиться переиспользовать код через наследование
- Освоить переопределение методов
- Реализовать полиморфное поведение

## Описание классов

### Базовый класс Server (base.py)

Базовый класс для всех серверов. Содержит общие атрибуты и методы.

| Атрибут | Тип | Описание |
|---------|-----|----------|
| _name | str | Имя сервера |
| _ip | str | IP-адрес |
| _active | bool | Активен/неактивен |
| _connections | int | Количество подключений |
| _status | str | Статус (Online/Offline/Maintenance) |
| _max_connections | int | Максимум подключений |

| Метод | Описание |
|-------|----------|
| ping() | Проверка доступности |
| connect() | Подключение к серверу |
| disconnect() | Отключение от сервера |
| restart() | Перезагрузка сервера |

---

### Дочерний класс GameServer (models.py)

Игровой сервер. Наследует всё от Server.

**Новые атрибуты:**
- `_game_type` — тип игры (FPS, RPG, Strategy и т.д.)
- `_max_players` — максимальное количество игроков

**Новый метод:**
- `start_match()` — начать матч

**Переопределённый метод:**
- `__str__()` — добавляет информацию об игре и игроках

---

### Дочерний класс DatabaseServer (models.py)

Сервер базы данных. Наследует всё от Server.

**Новые атрибуты:**
- `_db_type` — тип базы данных (PostgreSQL, MySQL, MongoDB)
- `_storage_gb` — объём хранилища в гигабайтах

**Новый метод:**
- `execute_query(query)` — выполнить запрос к БД

**Переопределённый метод:**
- `__str__()` — добавляет информацию о типе БД и объёме хранилища

---

### Класс ServerCollection (collection.py)

Коллекция для хранения серверов (из ЛР-2 с дополнениями).

**Новые методы для ЛР-3:**
- `get_by_type(server_type)` — фильтрация по типу
- `get_game_servers()` — вернуть только игровые серверы
- `get_database_servers()` — вернуть только серверы БД

## Демонстрация работы

В файле `demo.py` реализованы следующие сценарии:

1. **Создание объектов разных типов** — создание Server, GameServer, DatabaseServer
2. **Проверка наследования** — вызов методов родителя у дочерних объектов
3. **Новые методы дочерних классов** — start_match(), execute_query()
4. **Переопределённый __str__** — разный вывод для разных типов
5. **Работа с коллекцией** — добавление объектов разных типов в одну коллекцию
6. **Проверка типов через isinstance()** — определение реального типа объекта
7. **Фильтрация коллекции по типу** — get_game_servers(), get_database_servers()
8. **Полиморфизм** — одинаковый метод __str__ даёт разный результат
9. **Запрет дубликатов** — проверка уникальности IP
10. **Жизненный цикл** — полный сценарий работы с серверами
