## О проекте

Данная лабораторная работа является продолжением ЛР-1, ЛР-2 и ЛР-3. В проект добавлены **интерфейсы (абстрактные классы)**, которые задают контракты поведения для классов. Классы `Server`, `GameServer` и `DatabaseServer` реализуют эти интерфейсы, что обеспечивает полиморфное поведение и гарантирует наличие определённых методов.

---

## Цель работы

- Познакомиться с абстрактными базовыми классами (ABC)
- Освоить понятие интерфейса (контракта поведения)
- Научиться задавать обязательные методы для классов
- Закрепить полиморфизм через единый интерфейс
- Научиться проектировать архитектуру, а не просто классы

## Описание интерфейсов

### Интерфейс Printable (interfaces.py)

| Метод | Описание |
|-------|----------|
| `to_string() -> str` | Возвращает строковое представление объекта |

**Назначение:** гарантирует, что объект можно вывести в строку в удобном формате.

---

### Интерфейс Comparable (interfaces.py)

| Метод | Описание |
|-------|----------|
| `compare_to(other) -> int` | Сравнивает текущий объект с other. Возвращает -1, 0 или 1 |

**Назначение:** гарантирует, что объекты можно сравнивать между собой.

---

### Интерфейс Connectable (interfaces.py)

| Метод | Описание |
|-------|----------|
| `connect() -> str` | Подключиться к объекту |
| `disconnect() -> str` | Отключиться от объекта |
| `is_connected() -> bool` | Проверить наличие подключения |

**Назначение:** гарантирует, что к объекту можно подключаться и отключаться.

## Описание классов

### Базовый класс Server (base.py)

Реализует все три интерфейса. Содержит общие атрибуты и методы для всех серверов.

| Атрибут | Тип | Описание |
|---------|-----|----------|
| _name | str | Имя сервера |
| _ip | str | IP-адрес |
| _active | bool | Активен/неактивен |
| _connections | int | Количество подключений |
| _status | str | Статус (Online/Offline/Maintenance) |
| _max_connections | int | Максимум подключений |

**Реализация интерфейсов:**
- `to_string()` → возвращает `"Сервер {name}"`
- `compare_to()` → сравнивает по имени (алфавит)
- `connect()`, `disconnect()`, `is_connected()` → управление подключениями

---

### Дочерний класс GameServer (models.py)

Игровой сервер. Наследует всё от Server.

**Новые атрибуты:**
- `_game_type` — тип игры (FPS, RPG, Strategy)
- `_max_players` — максимальное количество игроков

**Новый метод:**
- `start_match()` — начать матч

**Переопределённые методы интерфейсов:**
- `to_string()` → возвращает `"[GAME] {name} | {game_type} | {players}/{max_players}"`
- `compare_to()` → сравнивает по количеству игроков

---

### Дочерний класс DatabaseServer (models.py)

Сервер базы данных. Наследует всё от Server.

**Новые атрибуты:**
- `_db_type` — тип базы данных (PostgreSQL, MySQL, MongoDB)
- `_storage_gb` — объём хранилища в гигабайтах

**Новый метод:**
- `execute_query(query)` — выполнить запрос

**Переопределённые методы интерфейсов:**
- `to_string()` → возвращает `"[DB] {name} | {db_type} | {storage_gb}GB"`
- `compare_to()` → сравнивает по объёму хранилища

---

### Класс ServerCollection (collection.py)

Коллекция для хранения серверов. Добавлены методы фильтрации по интерфейсу.

| Метод | Описание |
|-------|----------|
| `get_printable()` | Возвращает коллекцию объектов, реализующих Printable |
| `get_comparable()` | Возвращает коллекцию объектов, реализующих Comparable |

## Демонстрация работы

В файле `demo.py` реализованы следующие сценарии:

1. **Создание объектов разных типов** — Server, GameServer, DatabaseServer

9. **Сортировка коллекции** — sort_by_name()
10. **Сценарий мониторинга серверов**
11. **Запрет дубликатов** — проверка уникальности IP
