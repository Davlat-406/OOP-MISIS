## О проекте

Данная лабораторная работа является продолжением ЛР-1. Реализован контейнерный класс **ServerCollection** для хранения и управления объектами **Server** из первой лабораторной работы.

Коллекция позволяет добавлять, удалять, искать, сортировать и фильтровать серверы, а также поддерживает итерацию, доступ по индексу и срезы.

## Методы класса

### Управление коллекцией

| Метод | Описание |
|-------|----------|
| add(server) | Добавить сервер (с проверкой типа и дубликатов) |
| remove(server) | Удалить сервер |
| remove_at(index) | Удалить сервер по индексу (поддерживает отрицательные индексы) |
| get_all() | Получить список всех серверов |

### Поиск

| Метод | Описание |
|-------|----------|
| find_by_name(name) | Найти сервер по имени |
| find_by_ip(ip) | Найти сервер по IP |
| find_by_status(status) | Найти все серверы с определённым статусом |

### Сортировка

| Метод | Описание |
|-------|----------|
| sort_by_name() | Сортировка по имени (алфавит) |
| sort_by_ip() | Сортировка по IP |
| sort_by_connections() | Сортировка по количеству подключений |
| sort_by_status() | Сортировка по статусу (Online → Offline → Maintenance) |

### Фильтрация

| Метод | Описание |
|-------|----------|
| get_active_servers() | Вернуть только активные серверы |
| get_inactive_servers() | Вернуть только неактивные серверы |
| get_online_servers() | Вернуть только серверы со статусом Online |
| get_servers_with_connections(min, max) | Вернуть серверы с подключениями в диапазоне |

### Магические методы

| Метод | Описание |
|-------|----------|
| __len__() | Возвращает количество серверов |
| __iter__() | Позволяет итерировать коллекцию |
| __getitem__(index) | Доступ по индексу (поддерживает срезы и отрицательные индексы) |
| __str__() | Красивый вывод коллекции |
| __repr__() | Вывод для разработчиков |

## Демонстрация работы

![](/images/lab_02/geniral.png)

В файле `demo.py` реализованы следующие сценарии:

1. **Создание серверов** — создание 5 тестовых серверов
2. **Добавление в коллекцию** — добавление серверов в коллекцию
3. **Проверка запрета дубликатов** — попытка добавить сервер с существующим IP

5. **Поиск объектов** — find_by_name(), find_by_ip()
6. **Итерация и длина** — for server in collection, len()
7. **Доступ по индексу** — collection[0], collection[1:3], collection[-1]
8. **Сортировка** — sort_by_name(), sort_by_connections(), sort_by_status()
9. **Фильтрация** — get_active_servers(), get_online_servers()
10. **Удаление** — remove(), remove_at()
11. **Сценарий использования** — мониторинг серверов
12. **Работа с пустой коллекцией** — обработка граничных случаев
